# 테스트주도개발 정리

### 테스트주도개발 이란
#### TDD (Test-driven Development) <br>
 - 테스트를 먼저 작성하고 실제 코드 구현 <br>
 - 즉 테스트가 주도하여 개발하는 방식
 <br>

### TDD 흐름
기능을 검증하는 테스트 작성 -> 코딩 -> 리펙토링 -> 반복 <br><br>

### 작성순서
- 쉬운 경우 -> 어려운 경우 
- 예외적인 경우 -> 정상인 경우
  
해당 경우들을 순차적으로 작성 <br>
즉, 한번에 구현할 코드가 적은 순서로 작성 (구현하기 쉬움)
<br>

### 설계
#### 1. 기능
 - 입력
 - 결과

사용자에게 제공하는 기능은 위 두 가지로 나뉘어진다
<br>

#### 2. 이름 설계
- 다른 개발자가 봐도 한눈에 파악되도록 정확하게 이름 설계할 것
- 이 과정은 매우 중요한 부분
